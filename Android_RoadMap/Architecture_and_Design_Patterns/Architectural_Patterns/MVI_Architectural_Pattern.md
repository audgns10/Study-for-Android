# MVI - Architectural_Patterns

* JetPack Compose의 선언적 프로그램으 도입으로 인해 모든 안드로이드 개발에 인기있는 아키텍처 패턴중 하나임
* 다른 계층에 불변 상태를 제공하고 사용자의 작업 결과를 나타내며 UI 화면을 구성하는 상태의 단방향 및 불변성을 제공하는 단일 진실 원칙임
* MVVM, MVP와 달리 각 요소의 정의가 조금 다름

<br>

* Intent
  * 사용자의 작업(클릭 이벤트, UI 이벤트)을 처리하는 인터페이스 및 기능에 대한 정의임
  * 함수는 UI 이벤트를 Model의 인터페이스로 변환하고 결과를 반환할 수 있도록 Model에 전달을 함
  * 이름과 같이 Intent(의도) Model 함수을 실행하려는 의도라고 보면 됨
* Model
  * Intent의 출력을 가져와 View에서 랜더링 할 수 있는 UiState로 조작을 하는 기능적 메커니즘임
  * UiState는 변경할 수 없으며 단일 소스 및 단반향 데이터 흐름을 따르는 비즈니스 로직에서 가져옴
* View
  * View는 MVVM, MVP와 같은 역할을 함
  * 구현에서의 차이점은 MVI가 단방향 데이터 흐름을 보장하므로 View는 Model에서 오는 UI의 상태에 따라 UI 컴포넌트를 랜더링 하는 것임
